React & Vue (Konsep Modern)

Versi mudah dipahami â€” cocok untuk e-book, materi kuliah, atau ringkasan cepat tentang React dan Vue.

1. Component-Based Architecture

Baik React maupun Vue menggunakan komponen sebagai blok dasar aplikasi. Komponen adalah potongan UI kecil yang mandiri, memiliki logika sendiri, dan dapat digunakan kembali.

Contoh React (Functional)

<?jsx
function Hello() {
  return <h1>Hello World!</h1>;
}
?>

Contoh Vue (Single File / SFC)

<template>
  <h1>Hello World!</h1>
</template>

<script>
export default {
  name: "Hello"
};
</script>

2. Declarative UI

React dan Vue menggunakan pendekatan declarative â€” mendeskripsikan apa yang ingin ditampilkan, bukan bagaimana caranya. Pendekatan ini membuat kode lebih mudah dipahami dan dirawat.

3. Reactive Rendering (Reaktivitas)

Reaktivitas berarti UI akan otomatis berubah ketika data berubah.

React (state)

const [count, setCount] = useState(0);
setCount(count + 1);

Vue (reactive data)

data() {
  return { count: 0 };
},
methods: {
  increment() {
    this.count++;
  }
}

4. Virtual DOM

Virtual DOM adalah representasi ringan dari DOM asli. Ketika data berubah, framework menghitung diff dan hanya memperbarui bagian yang perlu diubah â€” membuat render lebih efisien.

5. Unidirectional Data Flow

Data mengalir dari parent ke child (satu arah) yang membuat alur data lebih terprediksi.

Vue menawarkan kemudahan tambahan melalui v-model untuk two-way binding pada input.

6. State Management (Modern)

Untuk aplikasi skala besar, gunakan solusi manajemen state yang terstruktur:

  • React: Context API, Redux, Zustand, Recoil
  • Vue: Pinia (modern), Vuex (lama)

7. Hooks (React) & Composition API (Vue)

Kedua konsep ini membuat logika komponen lebih modular dan dapat digunakan ulang.

React Hooks (contoh):

useEffect(() => {
  console.log("Mounted!");
}, []);

Vue Composition API (contoh):

<script setup>
import { ref } from 'vue';

const count = ref(0);
function increment() {
  count.value++;
}
</script>

Manfaatnya: menghindari class component, menulis logika yang reusable (hooks/composables), dan membuat kode lebih terstruktur.

8. Single File Component (SFC)

Satu file yang menggabungkan template, script, dan style. Umumnya dipakai di Vue, namun konsep serupa ada di React (mis. komponen dengan CSS module).

<template>
  <button>{{ count }}</button>
</template>

<script setup>
import { ref } from 'vue';
const count = ref(0);
</script>

<style>
button { padding: 10px; }
</style>

9. Ecosystem Modern

Tooling dan framework yang sering dipakai bersama:

  • React: Next.js (SSR & SSG), Remix, React Native
  • Vue: Nuxt (SSR & SSG), Quasar, Vue Native

Kesimpulan

React dan Vue adalah pilihan modern untuk membangun UI yang modular, reaktif, dan scalable. Pilih React jika butuh ekosistem besar dan banyak integrasi; pilih Vue jika menginginkan kemudahan penulisan dan struktur yang teratur.
